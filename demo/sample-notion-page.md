# My SaaS Idea — Design Asset Manager

## The Problem

I've been freelancing as a UI/UX designer for 3 years and the #1 thing that
slows me down is managing design assets across projects.

Every project I have:
- Figma files scattered across different team workspaces
- Assets exported to random folders on my desktop
- No way to quickly find "that button style I used 6 months ago"
- My clients ask for source files and I spend 30 mins hunting them down

## My Idea

A simple tool where designers can:
- Tag and organize exported design assets
- Search across all their projects
- Share asset libraries with clients in one click
- Maybe sync with Figma directly?

## What I've Done So Far

- Talked to 3 designer friends — they all said "omg yes I need this"
- Built a rough prototype in Notion (ironically) using a database + gallery view
- Started learning React to build the real thing

## The Problem With My Prototype

It works but it's clunky. I think I need to build a proper web app.
Maybe a Chrome extension? Or a desktop app?

I'm not sure where to start. The scope feels huge.

## Questions I Have

- Should I build the Figma plugin first or the web app?
- How do I handle file storage? AWS S3?
- What's the right pricing model — subscription or one-time?
- How do I find my first users?

## Notes

- My workaround right now: I export everything to a Google Drive folder with
  a very specific naming convention and then manually sort into subfolders.
  Takes me about 20 mins per project. Super annoying.
- There are tools like Eagle.app and Pastel but they don't really solve my
  specific workflow problem.
